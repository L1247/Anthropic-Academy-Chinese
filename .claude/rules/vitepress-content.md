# VitePress 內容規則

## 檔案與路由
- 檔案名稱使用英文小寫 kebab-case（如 `framework-foundations.md`），直接對應 URL slug
- 新增頁面後必須同步更新 `docs/.vitepress/config.mts` 的 sidebar，否則頁面存在但無法從導覽到達
- 新增課程後必須同步更新 `docs/index.md` 的課程總覽表格

## Markdown 規範
- 每頁只有一個 H1（`#`），作為頁面標題
- 使用 H2（`##`）和 H3（`###`）分層，config.mts 的 outline 設定會自動呈現 H2-H3 於目錄
- 難度標示統一格式：⭐ 初學者 / ⭐⭐ 中級 / ⭐⭐⭐ 高級

## Mermaid 圖表
- 直接使用 \`\`\`mermaid 語法塊，不需額外設定
- 主題已全域設定為 `default`，不在各頁面重複設定 `%%{init: ...}%%`
- 流程圖使用 `flowchart TD`（由上往下），節點文字超過 15 字時加換行 `<br/>`

## VitePress 元件
- 使用 `::: tip`、`::: info`、`::: warning` 標注重要提示
- 自訂 HTML（如角色卡片 `<div class="role-cards">`）與 `docs/index.md` 現有格式保持一致

## 並列清單的分行規則

VitePress 預設單一 `\n` 等同空格，不產生可見斷行。需在行末加**兩個空格**（`  `）才能觸發 `<br>` 硬換行。

**適用條件**：一行內出現 **2 個以上**的全形分號（；），代表有多個並列項目，必須分行顯示。

格式規則（`  ` 代表兩個空格）：

```
引導句（以 ：或 ——接清單）：  
項目 A——說明文字；  
項目 B——說明文字；  
項目 C——說明文字。  
補充說明句。
```

三類引導情境：

| 情境 | 引導方式 | 說明 |
|------|---------|------|
| 冒號引導 | 引導句末加 `  ` | `包含三類：  \n項目A；  \n項目B；` |
| 破折號接第一項 | 第一項在同行，其後分行 | `策略——項目A；  \n項目B；  \n項目C。` |
| 前置說明句 | 說明句末加 `  ` | `說明句。  \n項目A；  \n項目B；` |

不需分行的情況：
- 行內只有 1 個全形分號（；）
- 引號內的並列（用頓號 、分隔）
- Quiz 組件的 `explanation="..."` 字串屬性

## Blockquote 內換行規則

Blockquote（`>`）內的連續兩行若要各自單獨顯示（不被合併成同一段），**第一行末尾必須加兩個空格**（`  `）觸發 `<br>` 硬換行。

```markdown
<!-- ❌ 兩行會被合併成一段 -->
> 說明文字第一行。
> 說明文字第二行。

<!-- ✅ 正確：第一行末尾加兩個空格 -->
> 說明文字第一行。  
> 說明文字第二行。
```

**適用情境**：NLM 課程頁的頂部說明區塊（「以下內容由 NotebookLM 生成…」與「📖 回到主課程」兩行），以及任何需要在 blockquote 內顯示多行的情境。
