# 設計規範

## Emoji 使用規則

所有面向使用者的標題與提示區塊，必須包含 Emoji 以增強視覺識別性。

- **頁面標題（H1）**：必須有 Emoji，置於文字前（如 `# 🗺️ 學習路線圖`）
- **章節標題（H2/H3）**：視語意選用適合的 Emoji
- **VitePress 提示區塊標題**（`::: tip`、`::: info`、`::: warning`、`::: danger`）：標題文字必須附上 Emoji（如 `::: tip 🔍 圖表操作提示`）
- Emoji 置於標題文字**前方**，與文字之間保留一個空格

## 深色／淺色模式

所有 UI 元件、自訂 CSS 或視覺功能，必須同時考量深色模式（dark mode）與淺色模式（light mode）的顯示效果。

- 優先使用 VitePress 內建 CSS 變數（`var(--vp-c-*)` 系列），確保兩種模式自動切換
- 若需個別覆蓋，使用 `.dark` 選擇器提供深色模式對應樣式
- 新增元件後，在開發伺服器（`npm run docs:dev`）切換兩種模式驗證外觀
