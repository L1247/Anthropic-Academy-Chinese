# 🤝 Claude Cowork 入門

<Badge type="tip" text="⭐ 初學者" /> <Badge type="warning" text="完成可獲證書" />

> **原始課程**：[Introduction to Claude Cowork](https://anthropic.skilljar.com/introduction-to-claude-cowork)（英文）

## 📖 課程簡介

**Claude Cowork 是 Claude 直接在你的電腦上工作——讀取、編輯、產出真實的本地文件。**

這不是另一個聊天視窗。Cowork 讓 Claude 能夠操作你電腦上的真實檔案和應用程式：讀一份 PDF、修改一個 Excel、整理一個資料夾、跨應用程式彙整資訊——產出的結果會直接存在你的機器上。

> 「Chat 是一場對話，Cowork 是一次工作會議：你描述任務，Claude 規劃並執行，你在過程中引導方向。」

本課程從安裝到建立信心，帶你在第一週就成為 Cowork 的高效使用者。

## ⚠️ 前置條件

::: info 前置條件
**無需技術背景。** 任何能夠使用桌面應用程式的人都可以學習。建議先完成 [Claude 101](/claude-products/claude-101) 以熟悉 Claude 的基本互動模式。
:::

## 🎯 學習目標

完成本課程後，你將能夠：

- 理解 **Cowork 任務循環（Task Loop）** 的核心邏輯：描述 → Claude 規劃 → 執行 → 你引導
- 執行你的**第一個端到端任務**（從任務描述到實際檔案輸出）
- 了解**上下文如何影響 Claude 的規劃**，並善用這個特性
- 配置適合你工作的**插件（Plugins）和技能（Skills）**
- 執行**檔案工作流程**（讀取、整理、分析本地文件）
- 執行**研究工作流程**（蒐集、彙整多來源資訊）
- 掌握**引導長任務**的技巧，讓多步驟工作保持在正確軌道

## 📋 課程大綱

### 🤝 單元一：認識 Claude Cowork

建立 Cowork 的正確心智模型：

- **Cowork vs. claude.ai 網頁版**：Chat 是問答，Cowork 是任務執行
- **Cowork vs. Claude Code**：Cowork 面向所有人，Claude Code 面向開發者（程式碼和命令列工作）
- Cowork 最適合的場景：多步驟任務、涉及本地檔案、需要跨應用程式整合
- 安裝 Cowork 桌面應用程式（Mac / Windows）

### 🔄 單元二：Cowork 任務循環

Cowork 的核心工作模式——**任務循環（Task Loop）**：

```
1. 你描述任務目標（包含上下文、輸入、輸出、限制）
        ↓
2. Claude 規劃執行步驟（並向你確認）
        ↓
3. Claude 逐步執行（讀取檔案、處理資料、產出結果）
        ↓
4. 你在過程中觀察、給予回饋、調整方向
        ↓
5. 任務完成，結果存在你的電腦上
```

這堂課的關鍵學習：**上下文決定計畫的品質**。你提供的目標越清晰、背景越完整，Claude 的規劃就越準確，需要中途修正的次數就越少。

### ⚙️ 單元三：插件與技能設定

Cowork 的能力可以透過插件和技能擴展：

**插件（Plugins）：**
- 讓 Claude 連接和操作特定應用程式（例如：Notion、Google Docs、Slack 等）
- 需要授予對應的存取權限
- 管理原則：只授予當前任務需要的最小權限

<details>
<summary>詳細說明</summary>

插件的設定流程：在 Cowork 的設定介面中找到「Plugins」，選擇你需要的應用程式，點擊授權後完成 OAuth 認證。授權後，你就可以在任務描述中直接引用這個應用程式，例如「讀取我的 Notion 工作區中的『週報模板』頁面」。**最小權限原則**是關鍵：如果這個任務只需要「讀取」Notion，就不要授予「編輯」權限；任務結束後可以在設定中撤銷不再需要的插件存取。常見的實用插件組合：知識管理（Notion）+ 文件協作（Google Docs）+ 日曆（Google Calendar）。

</details>

**技能（Skills）：**
- 你自訂的、可重複使用的任務模板
- 例如：「每次整理我的會議記錄，請按照這個格式輸出，並自動標記待辦事項」
- 建立後可以一鍵觸發，不必每次重新描述

<details>
<summary>詳細說明</summary>

建立 Skill 的步驟：完成一次完整的任務後，把你覺得「下次還會再用同樣流程」的任務描述儲存為 Skill，給它一個清楚的名稱（例如「整理週報」）。下次執行時，點擊 Skill 名稱即可載入完整的任務描述，只需要替換其中的變數部分（例如本週的日期、檔案路徑），不必重新撰寫所有細節。一個好的 Skill 描述應該包含完整的輸入/輸出/限制說明，讓你每次執行都能得到一致品質的結果。

</details>

這堂課包含一個**插件設定練習**：根據你最常做的工作類型，選擇並設定 1–2 個最有幫助的插件。

### 📁 單元四：檔案工作流程

Cowork 最核心的使用場景——直接處理你電腦上的真實檔案：

**文件閱讀與分析：**
- 讓 Claude 讀取 PDF、Word、Excel，提取關鍵資訊
- 跨多份文件的比較分析

**批次處理：**
- 對資料夾中的多個檔案執行相同操作
- 例如：「讀取 reports/ 資料夾中所有 PDF，提取每份報告的日期、標題和主要結論，整理成一份 Excel 摘要表」

**跨應用程式資料流：**
- 從 Notion 取資料 → 整理後存到 Google Sheets
- 從 Email 附件 → 提取資訊 → 自動填入追蹤表格

### 🔍 單元五：研究工作流程

用 Cowork 處理需要多來源整合的研究任務：

- 指定多個資訊來源（網頁、本地文件、連結的應用程式）——在任務描述中明確列出每個來源的位置和形式，例如「讀取 Documents/research/ 資料夾中的 PDF，加上搜尋這些關鍵字的最新網頁資訊」。
- 自動化重複性的資訊蒐集和整理——對於每週或每月都需要執行的研究任務（例如競品動態追蹤、市場新聞彙整），把完整流程存成 Skill，定期一鍵執行。
- 建立主題知識庫：把零散資訊整理成結構化的參考文件——讓 Cowork 讀取多個來源後，以固定的結構（分類、標籤、摘要、原始連結）輸出到一份主控文件，逐次累積成可查詢的知識庫。

<details>
<summary>有效研究任務的描述架構</summary>

一個高效的研究任務描述包含四個要素：

1. **研究目標**：你要了解什麼問題？（例如：「比較 A、B、C 三個工具在價格和功能上的差異」）
2. **指定來源**：哪些本地文件 + 哪些要搜尋的關鍵字或網站
3. **輸出結構**：結果要用什麼格式呈現（比較表？摘要段落？分類列表？）
4. **輸出位置**：存到哪個資料夾、用什麼檔案名稱

描述越完整，Claude 越不需要猜測你的意圖，產出的品質也越穩定。

</details>

**範例任務：**
```
請幫我準備一份競品分析報告：
- 讀取 Documents/competitors/ 資料夾中我已存的 PDF 資料
- 補充搜尋這三家公司的最新產品更新（[公司A]、[公司B]、[公司C]）
- 以「功能比較表 + 各家最新動態摘要」格式輸出到桌面的 competitor_analysis.docx
```

### 🎯 單元六：引導長任務

多步驟任務容易偏離方向，這堂課教你掌握控制：

- **設定清晰的成功標準**：完成的任務應該長什麼樣子？
- **定義明確的邊界**：哪些資料夾可以讀取、哪些應用程式可以操作
- **定期檢查點**：每完成主要步驟後，確認方向再繼續
- **中途修正**：如何在任務進行中改變指示而不造成混亂
- **審查驗證**：任務完成後，如何系統性地確認結果品質

## 📝 重點筆記

### ⚖️ Cowork vs. claude.ai vs. Claude Code

| 產品 | 使用方式 | 最適合 |
|------|---------|--------|
| **claude.ai（網頁版）** | 瀏覽器聊天介面 | 單次問答、對話式任務、提示測試 |
| **Claude Cowork** | 桌面應用程式 | 處理本地檔案、多步驟任務、非技術使用者 |
| **Claude Code** | 命令列工具 | 開發者的程式碼工作、自動化腳本 |

### ✍️ 高品質任務描述的四要素

```
一個好的 Cowork 任務描述包含：

1. 目標：你要達成什麼？（具體說明最終結果）
2. 輸入：Claude 需要處理哪些檔案、資料或資訊來源？
3. 輸出：結果應該是什麼格式？存在哪裡？叫什麼名字？
4. 限制：有什麼不能做、不能存取、不能修改的？

範例：
「請讀取 Documents/Q1_reports/ 資料夾中所有 Excel 檔案，
提取每份的月份、部門和總銷售額，
整理成一個新的 Excel，儲存到桌面，命名為 Q1_summary.xlsx。
不要修改原始檔案。」
```

### 🎯 管理長任務的五個技巧

1. **先測試小範圍**：在一個或兩個檔案上測試，確認沒問題再擴大到整個資料夾
2. **先確認計畫**：讓 Claude 先說明它的執行計畫，你確認後再開始
3. **設定存取邊界**：告訴 Claude 哪些資料夾可以操作，哪些不行
4. **保留原始備份**：涉及修改檔案的任務，先確認有備份
5. **定期檢查點**：每個主要步驟完成後，檢查輸出是否符合預期

## 💡 學習建議

**實作練習：**

1. **第一個端到端任務**：安裝 Cowork 後，試著讓它讀取你桌面上的 3–5 份文件，產出一份整合摘要，觀察完整的任務循環是如何運作的。

2. **設定你的第一個 Skill**：找到你每週至少重複做一次的任務（例如：整理會議記錄、整理每日任務清單），把完整的任務描述存成一個 Skill，下週開始測試效率差異。

3. **高上下文任務練習**：選一個你這週真實需要完成的任務，先用「低上下文」描述（只說目標），觀察 Claude 的規劃；再用「高上下文」描述（加上輸入、輸出、限制），比較兩次規劃的差異。
