# 💬 Claude 101

## 📖 課程簡介

**如果你只能選一門課，選這門。**

Claude 101 是認識 Claude 最快速的入口課程，也是 Anthropic Academy 所有課程的必要基礎。課程帶你了解 Claude 的核心功能、實用工作流程和最佳實踐——讓你在最短時間內把 Claude 有效應用在日常工作中。

課程涵蓋：**提示技巧、Projects（專案功能）、Artifacts（產出物）、Skills（技能）、Connectors（連結器）、Research Mode（研究模式）**，以及如何選擇下一步的學習路徑。

無論你有沒有用過其他 AI 工具，這門課都能幫你掌握 Claude 的特色，快速上手。完成課程並通過測驗後，可取得 Anthropic 官方完成證書。

## 📋 課程大綱

### 💬 單元一：認識 Claude

- Claude 是什麼？它和其他 AI 助手有什麼不同？

  <details>
  <summary>詳細說明</summary>

  Claude 是由 Anthropic 開發的 AI 助手，能夠理解和生成文字、分析文件、撰寫程式碼，並在長達數萬字的對話中保持連貫的理解。與其他 AI 助手的差異在於：Claude 被設計為**優先誠實而非討好使用者**——當它不確定時會明說，而不是給出聽起來有信心但可能錯誤的答案。Anthropic 在訓練 Claude 時採用了「Constitutional AI（憲法 AI）」方法，讓 Claude 根據一套明確的原則來評估自己的回應，而非單純從人類回饋中學習「什麼答案受歡迎」。

  </details>

- Claude 的核心設計理念（Constitutional AI、誠實、無害）

  <details>
  <summary>詳細說明</summary>

  **Constitutional AI** 是 Anthropic 的訓練方法：透過一套明確的原則（類似「憲法」）讓模型自我評估和修正回應，使 AI 的行為更可預測、更符合人類價值觀。這帶來三個在使用上能感受到的特質：**誠實**（Claude 會承認自己不知道，不編造事實）、**無害**（拒絕協助可能造成傷害的請求，但不會過度謹慎地拒絕正當需求）、**有幫助**（盡力在安全的範圍內提供實用的回應）。

  </details>

- 如何在 claude.ai 建立帳號、開始第一次對話——前往 [claude.ai](https://claude.ai) 以 Google 帳號或電子郵件註冊；免費方案每天有使用次數限制，Pro 方案可取得更多額度和進階功能。建立帳號後直接在對話框輸入你的第一個問題即可開始。

- claude.ai 介面導覽——主畫面左側欄顯示所有過去的對話和 Projects；右上角可切換模型（Haiku/Sonnet/Opus）；對話框上方有上傳檔案按鈕，支援 PDF、Word、圖片等格式；部分回應右側會出現 Artifacts 面板，顯示獨立的程式碼或文件產出物。

**Claude 模型系列：**

claude.ai 提供三個模型層級，可從對話框的模型選單切換：

| 模型 | 定位 | 最適合的場景 |
|------|------|------------|
| **Claude Haiku** | 速度最快、最輕量 | 簡單問答、快速摘要、即時回應 |
| **Claude Sonnet** | 效能與速度平衡（預設） | 大多數日常工作任務 |
| **Claude Opus** | 推理能力最強 | 複雜分析、深度研究、多步驟推理 |

對大多數使用者來說，**Sonnet 是最佳預設選擇**；只有在處理高度複雜的推理任務時才需要切換到 Opus。

### ✍️ 單元二：有效提示的核心原則

好的提示是高品質輸出的基礎：

- **明確說明你要什麼**：輸出格式、長度、風格、受眾——不要只說「幫我寫一份報告」，而是「幫我寫一份 500 字以內的執行摘要，受眾是沒有技術背景的主管，使用條列式重點」。
- **提供充足的背景**：背景資訊越豐富，回應越精準——把相關脈絡、你的角色、任務的用途一起告訴 Claude，它才能做出符合你實際需求的判斷。
- **給予具體範例**：用實例展示你期望的輸出形式——如果你有過去的範例、想要的格式樣本，直接附上，Claude 會模仿其風格和結構。
- **迭代改進**：第一次不滿意？修改提示，重新對話——把 Claude 的輸出當成「初稿」，告訴它哪裡不對、需要如何調整，透過幾輪對話逐步收斂到你想要的結果。
- 使用「角色扮演」設定提升輸出品質——例如：「請以資深行銷顧問的角度，評估這份提案的市場定位策略」，能讓 Claude 切換到對應的思考框架，輸出更有針對性。

### 💼 單元三：日常工作應用

Claude 在這些場景表現最好：

**寫作與溝通：**
- 電子郵件草稿與回覆
- 報告、提案、簡報初稿
- 文章改寫與語氣調整

<details>
<summary>詳細說明與範例提示</summary>

Claude 的寫作能力最能發揮在「有明確目的的功能性寫作」，而非純粹的創意寫作。實用範例：

- **電子郵件**：「我需要婉拒一個合作邀請，但要維持良好關係。對方是我們的長期廠商。請幫我起草一封正式但友善的回絕信。」
- **簡報結構**：「這是我們Q3的業績數據（附上），請幫我整理成一份10頁簡報的大綱，對象是董事會，需要強調成長亮點但也誠實面對挑戰。」
- **語氣調整**：「把這段技術文件改寫成適合非技術主管閱讀的版本，去掉專業術語，保留核心訊息。」

</details>

**資料整理與分析：**
- 表格資料解讀與格式轉換
- 長文摘要與重點提取
- 問題整理與結構化

<details>
<summary>詳細說明與範例提示</summary>

Claude 在「將非結構化資訊轉換成結構化輸出」方面特別擅長。實用範例：

- **摘要**：「以下是本季的客戶回饋（貼上文字），請整理出前 5 個最常見的痛點，按照提及頻率排序。」
- **格式轉換**：「把這份清單轉成可以貼入 Excel 的 CSV 格式，欄位為：日期、項目名稱、金額、負責人。」
- **結構化整理**：「這是一段雜亂的會議討論記錄，請整理出：決議事項、待辦任務（含負責人）、未解決的問題。」

</details>

**文件處理：**
- 上傳 PDF、Word、Excel 分析
- 跨文件比較
- 合約或報告的重點標示

<details>
<summary>詳細說明與範例提示</summary>

Claude 支援直接上傳文件（拖曳到對話框），可處理 PDF、Word、Excel、圖片等格式。實用範例：

- **合約審閱**：「請閱讀這份合約（上傳 PDF），列出所有我方承擔的義務、關鍵的違約條款，以及你認為需要法律確認的模糊用語。」
- **跨文件比較**：「這是新舊兩份版本的服務條款（上傳兩份文件），請列出所有有實質差異的條款，並說明差異方向（更嚴格/更寬鬆）。」
- 注意：Claude 分析文件的結論仍需人工驗證，特別是法律和財務相關判斷。

</details>

### 📁 單元四：Projects——跨對話的長期記憶

Projects 是 claude.ai 的關鍵功能，解決「每次開新對話都要重新說明背景」的問題：

- 在 Project 中設定永久性的**系統指示**（你的角色、偏好、背景脈絡）
- 上傳**參考文件**（規格書、風格指南、過去的範例）
- 所有在同一個 Project 下的對話共享這些上下文
- 適用場景：長期客戶項目、重複性工作任務、個人助理設定

**範例：建立一個「行銷內容」Project**
```
系統指示：
「我是一家台灣 B2B 軟體公司的行銷主管。
我們的目標客群是中小企業的 IT 決策者。
回應請使用正式但不失親切的繁體中文，
避免過多技術術語。」

上傳的文件：
- 品牌風格指南.pdf
- 過去的成功案例文章.docx
```

### 🔧 單元五：Artifacts、Skills 與 Connectors

**Artifacts（產出物）：**
- Claude 生成的獨立內容塊（程式碼、文件、圖表等）
- 可以在對話旁邊預覽、直接編輯
- 支援 HTML/CSS 預覽、程式碼執行（部分功能）

<details>
<summary>詳細說明</summary>

當 Claude 生成一段較長的程式碼、一份完整的文件草稿，或一個可互動的元件時，這些內容會以「Artifact」的形式出現在對話右側的獨立面板中。你可以直接在 Artifact 面板中編輯內容、要求 Claude 修改特定部分，或點擊「複製」按鈕把完整內容取出使用。對於 HTML/CSS/JavaScript 產出，Artifact 還提供即時預覽，讓你不需要離開對話就能看到視覺效果。

</details>

**Skills（技能）：**
- 你可以建立並儲存的可重用指令集
- 例如：「每次整理會議記錄時，用這個格式輸出…」
- 減少重複輸入相同指示的時間

<details>
<summary>詳細說明</summary>

Skills 是你儲存在 claude.ai 中的自訂指令範本，一旦建立就可以在任何對話中一鍵呼叫。建立方式：在對話設定或 Project 設定中找到 Skills，新增一個有名稱的指令範本（例如名稱「整理會議記錄」，內容是完整的格式要求）。之後在對話中輸入 `@會議記錄` 就能快速載入這組指令，大幅減少每次重複輸入背景說明的時間。

</details>

**Connectors（連結器）：**
- 讓 Claude 連接外部服務（Google Drive、Confluence 等）
- 可以直接讀取你工作工具中的文件

<details>
<summary>詳細說明</summary>

Connectors 讓 Claude 能夠直接存取你的工作工具中的文件，不需要手動複製貼上。設定後，你可以在對話中說「請讀取我 Google Drive 中的 Q3 報告草稿並幫我改進摘要」，Claude 會直接取得最新版本的文件進行處理。目前支援的服務包含 Google Drive、Confluence 等，可在 claude.ai 的設定中管理授權。

</details>

### 🔍 單元六：Research Mode 與進階功能

**Research Mode：**
- 讓 Claude 自動在網路上搜尋資料，補充其知識截止日之後的最新資訊
- 適合需要最新數據、近期事件或即時資訊的查詢

<details>
<summary>詳細說明</summary>

啟用 Research Mode 後，Claude 會在生成回應前先自動執行多輪網路搜尋，蒐集最新資訊後再整合成有來源引用的完整答覆。這解決了 AI 模型「知識有截止日期」的根本限制——特別適合：查詢某公司最新財報、了解近期法規異動、確認某技術的最新版本或功能。使用時可以在對話框找到「Research」選項切換；回應中會列出搜尋來源，方便你追溯驗證。

</details>

**語音模式（Voice Mode）：**
- 用語音與 Claude 對話（適合行動裝置）——在 claude.ai 行動版點擊麥克風圖示啟用；支援即時語音輸入和語音回覆，適合邊走路邊思考、或不方便打字的場景。

**進階功能：**
- 長上下文視窗：一次處理大量文字（書本、法律文件等）——Claude 的上下文視窗可達 20 萬 Token（約 15 萬英文字），能一次讀取完整的長篇文件而不需要分批處理。
- 多輪對話：在同一對話中深入探討，Claude 會記住前面說過的內容——在同一個對話視窗中，Claude 保有完整的討論歷史，你可以直接說「剛才提到的第三點，請展開說明」，不需要重複背景。

## 📝 重點筆記

### 🎯 最有效的 Claude 使用場景

**高效場景（強烈推薦）：**
- 長文摘要與重點提取
- 寫作草稿與改寫優化
- 複雜概念的解釋與類比
- 程式碼說明與偵錯
- 資料整理與格式轉換

**中等效果（建議驗證）：**
- 數值計算（建議用其他工具確認）
- 需要最新資訊的查詢（使用 Research Mode 補充）

**不建議單獨依賴 AI 的場景：**
- 重大法律或醫療決策
- 需要高度精確事實引用的學術寫作

### 📧 提示範本：電子郵件寫作

```
請幫我寫一封電子郵件，內容如下：
- 收件人：[職稱/稱謂，例如：客戶、主管]
- 目的：[你想達成的目標，例如：感謝合作、詢問進度]
- 語氣：[正式 / 輕鬆 / 表達感謝]
- 需要包含的重點：
  1. [重點一]
  2. [重點二]
- 長度：[簡短（3–4 句）/ 中等（1–2 段）/ 完整]
```

### 📂 提示範本：文件分析

```
我已上傳一份[文件類型，例如：季度報告]，請幫我：
1. 用 5 個重點摘要這份文件的核心內容
2. 找出 3 個最值得關注的數字或趨勢
3. 列出任何你認為需要進一步了解的疑點
```

## 💡 學習建議

**實作練習：**

1. **第一週挑戰**：用 Claude 處理一封你這週收到的困難郵件——先自己想，再看 Claude 怎麼寫，比較差異。

2. **文件分析體驗**：上傳一份報告或會議記錄，要求 Claude 用 5 個重點摘要，觀察它的摘要重點和你的直覺是否一致。

3. **建立你的第一個 Project**：針對你最常做的一類工作（例如：撰寫技術文件、客戶溝通），建立一個 Project，加入系統指示和常用的參考文件，然後在 Project 內完成三個任務，感受有 Project 和沒有 Project 的差別。

- 同時搭配 [AI 能力與限制](/ai-fluency/capabilities-limitations) 了解 Claude 的邊界
