## 📖 課程簡介

**如果你只能選一門課，選這門。**

Claude 101 是認識 Claude 最快速的入口課程，也是 Anthropic Academy 所有課程的必要基礎。課程帶你了解 Claude 的核心功能、實用工作流程和最佳實踐——讓你在最短時間內把 Claude 有效應用在日常工作中。

課程涵蓋：**提示技巧、Projects（專案功能）、Artifacts（產出物）、Skills（技能）、Connectors（連結器）、Research Mode（研究模式）**，以及如何選擇下一步的學習路徑。

無論你有沒有用過其他 AI 工具，這門課都能幫你掌握 Claude 的特色，快速上手。完成課程並通過測驗後，可取得 Anthropic 官方完成證書。

完成本課程後，你將能夠：

## 📋 課程大綱

### 💬 單元一：認識 Claude

- Claude 是什麼？它和其他 AI 助手有什麼不同？
- Claude 的核心設計理念（Constitutional AI、誠實、無害）
- 如何在 claude.ai 建立帳號、開始第一次對話
- claude.ai 介面導覽

### ✍️ 單元二：有效提示的核心原則

好的提示是高品質輸出的基礎：

- **明確說明你要什麼**：輸出格式、長度、風格、受眾
- **提供充足的背景**：背景資訊越豐富，回應越精準
- **給予具體範例**：用實例展示你期望的輸出形式
- **迭代改進**：第一次不滿意？修改提示，重新對話
- 使用「角色扮演」設定提升輸出品質（例如：「請以資深行銷顧問的角度…」）

### 💼 單元三：日常工作應用

Claude 在這些場景表現最好：

**寫作與溝通：**
- 電子郵件草稿與回覆
- 報告、提案、簡報初稿
- 文章改寫與語氣調整

**資料整理與分析：**
- 表格資料解讀與格式轉換
- 長文摘要與重點提取
- 問題整理與結構化

**文件處理：**
- 上傳 PDF、Word、Excel 分析
- 跨文件比較
- 合約或報告的重點標示

### 📁 單元四：Projects——跨對話的長期記憶

Projects 是 claude.ai 的關鍵功能，解決「每次開新對話都要重新說明背景」的問題：

- 在 Project 中設定永久性的**系統指示**（你的角色、偏好、背景脈絡）
- 上傳**參考文件**（規格書、風格指南、過去的範例）
- 所有在同一個 Project 下的對話共享這些上下文
- 適用場景：長期客戶項目、重複性工作任務、個人助理設定

**範例：建立一個「行銷內容」Project**
```
系統指示：
「我是一家台灣 B2B 軟體公司的行銷主管。
我們的目標客群是中小企業的 IT 決策者。
回應請使用正式但不失親切的繁體中文，
避免過多技術術語。」

上傳的文件：
- 品牌風格指南.pdf
- 過去的成功案例文章.docx
```

### 🔧 單元五：Artifacts、Skills 與 Connectors

**Artifacts（產出物）：**
- Claude 生成的獨立內容塊（程式碼、文件、圖表等）
- 可以在對話旁邊預覽、直接編輯
- 支援 HTML/CSS 預覽、程式碼執行（部分功能）

**Skills（技能）：**
- 你可以建立並儲存的可重用指令集
- 例如：「每次整理會議記錄時，用這個格式輸出…」
- 減少重複輸入相同指示的時間

**Connectors（連結器）：**
- 讓 Claude 連接外部服務（Google Drive、Confluence 等）
- 可以直接讀取你工作工具中的文件

### 🔍 單元六：Research Mode 與進階功能

**Research Mode：**
- 讓 Claude 自動在網路上搜尋資料，補充其知識截止日之後的最新資訊
- 適合需要最新數據、近期事件或即時資訊的查詢

**語音模式（Voice Mode）：**
- 用語音與 Claude 對話（適合行動裝置）

**進階功能：**
- 長上下文視窗：一次處理大量文字（書本、法律文件等）
- 多輪對話：在同一對話中深入探討，Claude 會記住前面說過的內容

## 📝 重點筆記

### 🎯 最有效的 Claude 使用場景

**高效場景（強烈推薦）：**
- 長文摘要與重點提取
- 寫作草稿與改寫優化
- 複雜概念的解釋與類比
- 程式碼說明與偵錯
- 資料整理與格式轉換

**中等效果（建議驗證）：**
- 數值計算（建議用其他工具確認）
- 需要最新資訊的查詢（使用 Research Mode 補充）

**不建議單獨依賴 AI 的場景：**
- 重大法律或醫療決策
- 需要高度精確事實引用的學術寫作

### 📧 提示範本：電子郵件寫作

```
請幫我寫一封電子郵件，內容如下：
- 收件人：[職稱/稱謂，例如：客戶、主管]
- 目的：[你想達成的目標，例如：感謝合作、詢問進度]
- 語氣：[正式 / 輕鬆 / 表達感謝]
- 需要包含的重點：
  1. [重點一]
  2. [重點二]
- 長度：[簡短（3–4 句）/ 中等（1–2 段）/ 完整]
```

### 📂 提示範本：文件分析

```
我已上傳一份[文件類型，例如：季度報告]，請幫我：
1. 用 5 個重點摘要這份文件的核心內容
2. 找出 3 個最值得關注的數字或趨勢
3. 列出任何你認為需要進一步了解的疑點
```

## 💡 學習建議

**實作練習：**

1. **第一週挑戰**：用 Claude 處理一封你這週收到的困難郵件——先自己想，再看 Claude 怎麼寫，比較差異。

2. **文件分析體驗**：上傳一份報告或會議記錄，要求 Claude 用 5 個重點摘要，觀察它的摘要重點和你的直覺是否一致。

3. **建立你的第一個 Project**：針對你最常做的一類工作（例如：撰寫技術文件、客戶溝通），建立一個 Project，加入系統指示和常用的參考文件，然後在 Project 內完成三個任務，感受有 Project 和沒有 Project 的差別。
